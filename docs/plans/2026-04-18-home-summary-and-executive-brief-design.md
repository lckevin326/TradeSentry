# 首页综合摘要与经营摘要增强设计

**目标**

在现有“决策建议”模块基础上，同时补齐两层能力：

1. 把规则驱动建议升级成更贴近管理层阅读的“经营摘要”
2. 把首页从分散的信息卡，升级成“今日出口环境综合摘要 + 决策建议 + 明细趋势”的结构

**产品判断**

这一轮不接 AI 做判定，也不把规则引擎变成黑盒。AI 仍然只作为后续可选的解释层。本轮所有结论仍由结构化规则和模板输出。

**范围**

本轮只做：

- 扩充 `decisionAdvice` 的模板输出层，增加更像老板口径的摘要
- 首页新增一个综合摘要区，汇总汇率、运费、政策、利润建议
- 用现有数据源生成首页摘要，不新增外部抓取源

本轮不做：

- AI 模型接入
- 多市场横向比较
- 晨报/周报投递
- 新的数据采集任务

## 一、经营摘要层设计

现有 `decisionAdvice` 已经有：

- `status`
- `summary`
- `dominantDriver`
- `action`
- `warnings`
- `explanation`

但它还偏“规则结果”。这一轮需要把它补成“可直接给老板看”的结构。

### 新增输出

建议在 `DecisionAdvice` 中增加：

- `executiveSummary: string`
- `driverBreakdown: Array<{ label: string; impactCny: number; tone: 'positive' | 'negative' | 'neutral' }>`

其中：

- `executiveSummary` 是 1-2 句经营口径，不是重复 `summary`
- `driverBreakdown` 给首页综合摘要区和卡片副区块复用

### 生成规则

`executiveSummary` 仍由模板生成，组合以下信息：

- 当前状态：健康 / 关注 / 承压
- 较昨日利润变化：改善 / 收缩 / 持平
- 主驱动：汇率 / 运费 / 税费 / 多因子
- 当前动作：维持报价 / 复核运费 / 上调报价 / 人工复核

示例口径：

- 当前报价环境偏健康，利润较昨日改善，主要受汇率走强带动，短期可维持当前报价。
- 当前利润进入承压区间，主要压力来自运费抬升，建议先复核运费口径，再决定是否上调报价。
- 当前变化由多因子共同驱动，暂不宜只依据单一参数调整报价，应结合最新政策与运费一并判断。

`driverBreakdown` 直接基于 attribution 生成，不再引入第二套算法。

### 空态原则

当没有足够测算数据时：

- 显示占位卡片
- 展示“完成测算后生成建议”
- 不展示状态 badge
- 不展示 driver/action/warnings/executiveSummary

## 二、首页综合摘要区设计

首页顶部指标卡目前仍是四个独立卡片。它们信息密度够，但还没有形成“今日结论”。

这一轮在顶部指标卡之后、决策建议卡之前新增一个“今日出口环境摘要”模块。

### 结构

模块包含三块：

1. 综合结论
2. 三条关键信号
3. 当日关注项

### 1. 综合结论

以一句话回答：

- 今天整体环境是偏有利、偏中性还是偏承压

判定优先级：

- 若 `decisionAdvice.status === 'pressure'`，首页环境摘要为“偏承压”
- 若 `decisionAdvice.status === 'watch'`，为“偏中性”
- 若 `decisionAdvice.status === 'healthy'`，为“偏有利”
- 若无建议数据，则显示“等待测算”

### 2. 三条关键信号

从现有可用数据中提炼三条：

- 汇率信号：AED 当日变化
- 运费信号：最新运费与近一期对比
- 政策信号：最近政策数量和最近一条标题

输出形式是短句，不是图表：

- 汇率小幅走强，报价折算压力减轻
- 中东默认运费较上一期回落 / 抬升
- 最新政策 1 条，需关注税费口径是否同步更新

### 3. 当日关注项

当有 `decisionAdvice.warnings` 时，取前 1-2 条作为首页关注项；
若没有 warnings，则回退为：

- 暂无额外风险提示

## 三、首页布局调整

新的首页顺序：

1. 标题区
2. 顶部指标卡
3. 今日出口环境摘要
4. 决策建议
5. 汇率走势 + 预警中心
6. 利润试算 / 利润摘要 / 归因
7. 运费趋势 + 政策时间轴

这样首页阅读顺序会变成：

- 先看今天环境
- 再看系统建议
- 再看证据细节

## 四、实现约束

- `DecisionAdvice` 的生成逻辑仍只在 `lib/profit` 下
- 页面层只能消费结果，不再拼第二套业务判断
- 客户端重新测算后，必须复用同一个共享 helper 刷新 summary 和 advice
- 所有新增 UI 必须有空态测试
- 不引入新的接口请求

## 五、成功标准

- 首页出现“今日出口环境摘要”模块
- 决策建议卡有更清晰的经营摘要口径
- 页面无建议数据时显示明确空态，不误导
- 首页读序更像“结论 -> 建议 -> 证据”
- 所有现有测试继续通过
